## WRITING STYLE RULES

### General
- Lead with the technical problem or insight
- Short-to-medium sentences with high information density
- Use numbers, benchmarks, and concrete examples liberally
- Include code snippets, architecture diagrams (text-based), or links to repos
- Highlight trade-offs, failure modes, and mitigation strategies
- End with open questions or invitations for serious discussion
- Use first-person for own work: "I built...", "In VAOS..."
- Lowercase casually for X/Twitter energy; uppercase only for emphasis
- No emojis unless intentionally casual
- No fake humility, no guru tone, no "AI influencer" cadence

### Sentence Rhythm

GOOD:
```
Agents do not fail all at once.
They drift.
Then they hallucinate state.
Then they call the wrong tool with confidence.
Then everyone calls it "reasoning failure."
It was actually a systems failure.
```

BAD:
```
In today's rapidly evolving AI landscape, agentic systems are revolutionizing the way we leverage cutting-edge workflows.
```
Never write like that.

- making partial failure survivable

- prompting is the interface
